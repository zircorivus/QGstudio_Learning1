## DX11：纹理和UV
### 从UV到纹理
模型在建造完成后，经过展UV后，模型的顶点就会绑定上一个UV二维坐标，这个坐标对应的就是贴图上面的坐标点，在像素着色器中根据顶点的插值确认对应的贴图颜色，就能将模型表面贴上纹理。
1. 在顶点结构体里面添加TEXCOORD语义的成员结构体，用于装载UV坐标
2. 将纹理文件加载到GPU。有专门的加载用的函数，需要在里面写入指定贴图的文件位置，就能将静态的文件加载进来了。
    * DDS格式：DirectDraw Surface，是DirectX纹理压缩（DXTC）的原生格式，支持块压缩、mipmap链，加载速度快，内存占用小，是游戏中的标准格式。
    ![示例图](屏幕截图%202026-03-30%20170643.png)
    * WIC：Windows Imaging Component，可以加载常见的位图格式（bmp, jpg, png等），便于开发调试时使用普通图片。
    ![示例图](屏幕截图%202026-03-30%20170656.png)
3. 采样器状态
    1. 过滤器：点采样/线性采样/各向异性采样
    2. 寻址模式：当纹理坐标超出[0,1]范围时处理的方式
    3. 创建采样器并绑定：就像前面的常量缓冲区之类的，创建描述之后，要对里面的成员进行设定，然后再绑定到D3D设备上面
4. 再与光照模型进行结合。一般纹理会影响漫反射和环境光，一般高光颜色不受影响。   
综上所述，其实UV贴图的知识大概就是这样

## 混合模式的天坑
承上启下的关键一章。
### 在像素着色器之后
你还记得我们之前画的渲染管线流程图吗？顶点着色器 → 光栅化 → 像素着色器 → 输出合并阶段（OM） → 后台缓冲区。

像素着色器输出的是一个颜色值（RGBA），但此时它还没有真正写入后台缓冲区。在输出合并阶段，GPU会进行一系列测试和操作：

裁剪测试（Scissor Test）

深度测试（Depth Test）

模板测试（Stencil Test）

混合（Blending）

只有当这些测试都通过后，像素才会被写入渲染目标（通常是后台缓冲区）。混合就是其中最后一个可编程步骤，它决定了当前像素的颜色如何与缓冲区中已有的像素颜色叠加。
### 混合的核心公式
关键点：混合是逐颜色分量进行的，并且颜色和 Alpha 可以分别控制。
* 最终颜色 = 源颜色 × 源因子 + 目标颜色 × 目标因子
### 混合因子（Blend Factor）—— 控制源和目标的影响程度
混合因子决定了源像素和目标像素各自对最终颜色的贡献权重。
### 混合运算符（Blend Operation）
⊞ 不只是加法，还可以是减法、反向减法、取最小值、取最大值。文档中的 D3D11_BLEND_OP 枚举提供了这些选项。例如：

D3D11_BLEND_OP_ADD：最常用。

D3D11_BLEND_OP_SUBTRACT：C_dst - C_src。

D3D11_BLEND_OP_MIN：取两个颜色的最小值。

应用场景：减法可以用来做“污渍”效果（减色），MIN 可以模拟多层阴影叠加。
### 混合状态的创建与设置
在 Direct3D 11 中，混合状态是一个 状态对象，需要先填充描述结构体 D3D11_BLEND_DESC，然后调用 CreateBlendState 创建，最后在渲染时用 OMSetBlendState 绑定。
1. AlphaToCoverageEnable：用于多重采样抗锯齿，开启后可以将 Alpha 值转换为采样覆盖率，改善透明物体的边缘锯齿。一般用于粒子系统。
2. IndependentBlendEnable：是否允许对多个渲染目标使用不同的混合设置。默认 false，只用第一个。
3. BlendEnable：是否开启混合（不开启则直接覆盖）
4. SrcBlend、DestBlend、BlendOp：颜色混合因子和运算符
5. SrcBlendAlpha、DestBlendAlpha、BlendOpAlpha：Alpha 通道的混合因子和运算符
6. RenderTargetWriteMask：允许写入哪些颜色通道（例如只写红色通道可制作特殊效果）   



Direct3D 是基于状态机的，这些状态对象可以提前创建，在绘制不同物体时切换。
### 透明混合的陷阱：深度测试与绘制顺序
必须先绘制不透明物体，然后按从远到近的顺序绘制透明物体。

为什么？因为深度测试会丢弃被遮挡的像素。如果你先绘制一个近处的半透明物体，它的深度值会写入深度缓冲区。之后远处的物体因为深度值更大（更远），无法通过深度测试，就不会被绘制，导致透明物体背后的东西消失。

正确的做法：

1. 绘制所有不透明物体（开启深度写入和深度测试）。

2. 对透明物体按距离摄像机从远到近排序。

3. 绘制透明物体时，可以继续开启深度测试（防止透明物体之间互相穿透），但通常要关闭深度写入（否则远处透明物体会挡住近处透明物体）。具体看需求，一般半透明物体不需要写入深度，但需要测试深度。
### 常量缓冲区的进一步拆分
文档中还将常量缓冲区按更新频率拆分成了四个：
* CBChangesEveryDrawing：每绘制一个物体都可能改变（世界矩阵、材质）
* CBChangesEveryFrame：每帧改变（观察矩阵、眼睛位置）
* CBChangesOnResize：窗口大小改变时（投影矩阵）
* CBChangesRarely：几乎不变（光源、场景配置）
## 一些关键名词的辨析
1. 深度缓冲，每个像素在被成功绘制之前，就会先计算一遍深度，先存起来，等着后面深度测试再使用
2. 深度测试，比较当前这个像素的深度和准备绘制的缓冲区里面的像素的深度，要么保留，要么放弃这个像素
3. 混合，利用公式，将当前像素的颜色和已经在缓冲区里面的像素的颜色进行混合
4. 混合因子，计算公式的一部分，决定两种颜色各自占多少部分
5. 混合状态，就是从像素着色器里面出来的像素和缓冲区里面的像素怎么混合的综合状态。
## 混合状态的辨析
在Xjun的项目里面，专门设置了一个公共类对渲染的混合状态进行了封装，避免了直接访问DX11冗长的API设置，并且方便切换。先在头文件里面定义各种状态变量的名字。
1. 光栅化器状态
    * 线框模式RSWireframe
    * 无背面裁剪模式RSNoCull
    * 顺时针裁剪模式RSCullClockWise
2. 采样器状态
    * 线性过滤SSLinearWrap
    * 各项异性过滤SSAnisotropicWrap
3. 混合状态
    * 不写入颜色BSNoColorWrite
    * 透明混合BSTransparent
    * Alpha-To-Coverage
    * 加法混合BSAdditive
4. 深度/模板状态
    * 写入模板值DSSWriteStencil
    * 对指定模板值的区域进行绘制DSSDrawWithStencil
    * 无二次混合区域DSSNoDoubleBlend
    * 关闭深度测试DSSNoDepthTest
    * 仅深度测试，不写入深度值DSSNoDepthWrite
    * 关闭深度测试，对指定模板值的区域进行绘制DSSNoDepthTestWithStencil
    * 仅深度测试，不写入深度值，对指定模板值的区域进行绘制DSSNoDepthWriteWithStencil


然后在cpp文件里面，将他们设为静态成员，并用智能指针访问，在下面的初始化函数里面，用临时变量设置好一个模式里面的各个变量，然后用DX11的原生API进行一个状态的生成。因为这个状态的变量是静态的，所以可以全局访问，可以随时切换。
## 搞清深度写入和深度测试
先分清楚两个名词。深度写入：将绘制出来的像素，在进入帧缓冲区之前，将像素的深度写入深度缓冲区（一个屏幕像素量的数组，和什么常量缓冲区没有关系）。深度测试：新绘制一个像素，如果要进行深度测试就是要测试是不是比已经在深度缓冲区里面的像素更加靠后，如果更加靠后，就不会绘制了。

所以半透明的物体，要打开深度测试，但是不能打开深度写入，因为半透明物体会被不透明物体挡住，但是半透明物体不会挡住背后的物体，所以不能将他的深度写入深度缓冲区。

## 混合状态的进一步理解
混合状态描述结构体D3D11_BLEND_DESC就是用来设置混合状态的。我们之前说的什么透明混合，加法混合，都只是对其中成员设置组合的经验惯例。

对于混合状态描述结构体，其中含有至多8个的渲染目标混合描述，可以分别设置不同对象的渲染方式。

typedef struct D3D11_BLEND_DESC 
{
*   BOOL AlphaToCoverageEnable;                // 当开启多重采样（MSAA）时，将Alpha值转换为采样覆盖率，改善透明纹理边缘抗锯齿。通常不开启。
* BOOL IndependentBlendEnable;               // 设为 FALSE 则所有渲染目标都使用 RenderTarget[0] 的设置；若设为 TRUE，则每个渲染目标可以有不同的混合描述（最多8个）。
* D3D11_RENDER_TARGET_BLEND_DESC RenderTarget[8]; // 每个渲染目标的混合描述，通常只用[0]

} D3D11_BLEND_DESC;

在其中的数组成员：渲染目标混合描述 D3D11_RENDER_TARGET_BLEND_DESC

typedef struct D3D11_RENDER_TARGET_BLEND_DESC    
{
* BOOL BlendEnable;                 // 是否启用混合（若为FALSE，则直接覆盖）
* D3D11_BLEND SrcBlend;            // 源颜色混合因子（RGB）混合因子枚举
    * D3D11_BLEND_ZERO
    * D3D11_BLEND_ONE
    * D3D11_BLEND_SRC_COLOR
    * D3D11_BLEND_INV_SRC_COLOR
    * D3D11_BLEND_SRC_ALPHA
    * D3D11_BLEND_INV_SRC_ALPHA	
    * D3D11_BLEND_DEST_ALPHA
    * D3D11_BLEND_INV_DEST_ALPHA
    * D3D11_BLEND_DEST_COLOR
    * D3D11_BLEND_INV_DEST_COLOR
    * D3D11_BLEND_SRC_ALPHA_SAT
    * D3D11_BLEND_BLEND_FACTOR
    * D3D11_BLEND_INV_BLEND_FACTOR
    ![示例](屏幕截图%202026-04-01%20180150.png)	
* D3D11_BLEND DestBlend;           // 目标颜色混合因子（RGB）
* D3D11_BLEND_OP BlendOp;          // 颜色混合运算符，混合运算符因子
    * D3D11_BLEND_OP_ADD
    * D3D11_BLEND_OP_SUBTRACT
    * D3D11_BLEND_OP_REV_SUBTRACT
    * D3D11_BLEND_OP_MIN
    * D3D11_BLEND_OP_MAX
    ![示例](屏幕截图%202026-04-01%20180204.png)

* D3D11_BLEND DestBlendAlpha;      // 目标Alpha混合因子
* D3D11_BLEND_OP BlendOpAlpha;     // Alpha混合运算符
* UINT8 RenderTargetWriteMask;     // 允许写入的颜色通道（位掩码）
    * D3D11_COLOR_WRITE_ENABLE_RED写入红色通道
    * D3D11_COLOR_WRITE_ENABLE_GREEN写入绿色通道
    * D3D11_COLOR_WRITE_ENABLE_BLUE写入蓝色通道
    * D3D11_COLOR_WRITE_ENABLE_ALPHA写入Alpha通道
    * D3D11_COLOR_WRITE_ENABLE_ALL写入所有通道（=0xF）

## Billn-phong模型
这个效果在第二周的作业里面已经手搓实现完成了，还有增加了简单的粗糙度和能量守恒效果。billn-phong模型的关键不同就在于从计算光线向量和视线向量的夹角变成了计算理想法线（半角向量）和实际法线的夹角。这个夹角影响的只是高光，环境光是不用计算角度的，漫反射就是直接用的夹角余弦。
* 实际的计算公式里面是材质的float4乘上光源的float4，再去乘别的因子，这样子的设计使得float4可以模拟材质反射颜色的物理性质。
* 粗糙度的引入，就是使漫反射的因子计算变成了另一套公式，但是我不记得了...但是仍然存在比较严重的硬伤，粗糙度较低时高光面积极小且不明显，粗糙度较高时镜面反射几乎消失，镜面反射范围极大但明暗交界线生硬
* 简单的能量守恒就是：假设漫反射的因子是x，那么镜面反射的因子就是(1-x)，实现的效果比较简陋
* 通过公式对镜面反射的高光进行柔化边缘，改善粗糙度较大时明暗过渡生硬的问题。
