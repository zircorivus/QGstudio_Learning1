## 常量缓冲区的更新方式
1. 使用常量缓冲区的原因：顶点着色器和片元着色器需要访问外部数据，作为频繁更新的信息，每次计算都单独传输顶点信息，会产生大量数据传输开销，还有频繁内存寻址的开销。所以每次传输一次性传输一批数据，着色器可以快速访问，而不再需要频繁访问外部。
2. 一般的更新方式：在CPU端创建一个结构体，调用API将其整个可以拷贝到显存。就比如UnityShader里面出现的CBuffer，应该就是类似的部分
    * 缺点：每次更新重新上传或分配整个缓冲区，CPU与GPU之间存在同步的隐患，可能引发流水线停顿
    * D指导专业解释：CPU 每帧调用 UpdateSubresource (Direct3D 11) 或 glBufferData (OpenGL) 函数，将整个常量缓冲区的新数据从系统内存拷贝到显存。
3. 预分配+部分更新：预先分配好缓冲区不同的区域，然后用类似 glBufferSubData 或 UpdateSubresource 只更新需要的区域。
    * 缺点：仍可能触发 CPU-GPU 同步，GPU 可能还在使用该缓冲区的旧数据，而 CPU 直接修改会造成数据争用。
4. 使用映射直接写入：在上述两种更新方式里，使用完一次常量数据，就要重新分配一遍结构体的内存。使用映射的方法，将显存地址直接提交给CPU，由CPU直接上传，避免了重新分配，但是CPU与GPU仍可能出现读写冲突。
    * D指导专业解释：CPU 调用 Map 方法获取指向 GPU 内存（常量缓冲区）的指针，直接写入新数据，然后调用 Unmap 释放指针。
5. 持久映射？将常量缓冲区的地址持续开放给CPU，CPU直接上传数据。同时通过“围栏（Fence）”或信号量保证两边不会同时访问造成读写冲突。
    * D指导专业解释：主要在 Direct3D 12 / Vulkan 这类底层 API 中使用。创建资源（如上传堆）时指定为 持久映射 (Persistent Mapping)，得到一个 CPU 长期有效的指针。之后每次更新数据直接用 memcpy 写入，不再反复 Map/Unmap。
6. 环形缓冲区与多重缓冲：将常量缓冲区分成的多个部分，CPU依次读入，GPU依次读取，保证了并行处理的时效性。
    * D指导专业解释：在 CPU 和 GPU 之间设置一个较大的缓冲区，逻辑上划分为 N 个槽位 (Slots)（例如 N=3）。CPU 按顺序写入当前槽位，写入后通知 GPU 可以读取该槽位，然后立即转到下一个槽位继续写入。GPU 也按顺序读取槽位。

## 图元，片元和像素的区别
* 图元是GPU渲染过程中最小的用于渲染的几何图形，比如圆，三角形，矩形。最常见的就是模型上的三角面
    * D指导的修改意见，图元是不可再细分的几何形状，是光栅化阶段的基本输入单元
* 片元是包含像素信息在内的，很多状态信息的集合，包含了屏幕坐标，深度信息，法线信息，纹理信息
    * D指导：片元不包含像素，片元是各个像素位置的候选者，经过一系列测试后才能决定能否转换为像素
* 渲染好的像素才是最终会呈现到屏幕上的信息，但从渲染管线离开后，可能还要进入画面后处理阶段
    * D指导：最终写入帧缓冲区的像素已经经过所有测试和颜色混合，但后处理阶段（如色调映射、抗锯齿、HDR合成）可能会在像素写入后再次读取并修改它们，所以“最终呈现”有时是多遍渲染的结果。

## DX11的学习

### 索引缓冲区
今天学的是第三章的常量缓冲区和索引缓冲区，索引的作用就是复用顶点坐标，减小顶点数组体积。新建索引缓冲区、常量缓冲区和新建顶点缓冲区的方法都是相似的。

`// 设置索引缓冲区描述`

`D3D11_BUFFER_DESC ibd;`

`ZeroMemory(&ibd, sizeof(ibd));`  

`ibd.Usage = D3D11_USAGE_IMMUTABLE;`  

`ibd.ByteWidth = sizeof indices;`

`ibd.BindFlags = D3D11_BIND_INDEX_BUFFER;`

`ibd.CPUAccessFlags = 0;`

`// 新建索引缓冲区`
`InitData.pSysMem = indices;`

`HR(m_pd3dDevice->CreateBuffer(&ibd, &InitData, m_pIndexBuffer.GetAddressOf()));`

 * D3D11_BUFFER_DESC 是缓冲区的结构体类型。
 * ZeroMemory应该是初始化内存
 * Usage是指明这个缓冲区的读写更新方式，
 * immutable表示仅初始化一次，不再更新。 
 * ByteWidth是缓冲区的长度。
 * BindFlags是指明这是哪一种缓冲区。
 * CPUAsseccFlags是CPU的访问权限。
